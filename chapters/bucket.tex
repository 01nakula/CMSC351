\section{Bucket Sort}
\rule{\textwidth}{1pt}\\
\subsection{Pseudocode}
Note: Assume you have an uniformly distributed array of real numbers
between 0 and 1. Create $n$ buckets.
\begin{algorithm}
\caption{Bucket Sort}
\begin{algorithmic}[1]
\State \textbf{for} $i=0$ \textbf{to} $n-1$ \textbf{do} $B[i] \leftarrow 0$
\Comment Clear $B$
\Statex
\For{$i=1$ \textbf{to} $n$}
	\State $B[\lfloor n\cdot A[i]\rfloor] \leftarrow A[i]$
\EndFor
\Statex
\State Sort $B$ \Comment Can be done with any sorting algorithm
\State Concatenate $B$
\end{algorithmic}
\end{algorithm}

\subsection{Analysis}
On average there will be one item per bucket, and
lucky for us it is unlikely that a bucket will have a lot of 
elements in it. So average case, bucket sort is $O(n)$ or \textit{linear} time.

\subsection{Modifications}
\subsubsection*{Between Two Real Numbers}
What if we want to sort numbers between $[q, r)$?
We can simply renormalize $B$ like so.\\
\rule{\textwidth}{.5pt}\\
\begin{algorithmic}[1]
\State \textbf{for} $i=0$ \textbf{to} $n-1$ \textbf{do} $B[i] \leftarrow 0$
\Comment Clear $B$
\Statex
\For{$i=1$ \textbf{to} $n$}
	\State $B\left[\left\lfloor  \frac{n\cdot A[i]}{r - q} \right\rfloor\right] \leftarrow A[i]$
\EndFor
\Statex
\State Sort $B$ \Comment Can be done with any sorting algorithm
\State Concatenate $B$
\end{algorithmic}
\rule{\textwidth}{.5pt}\\
\subsubsection*{Nonuniform Distribution}
What if the data is not uniformly distributed?
We can set the buckets' spread larger and larger as it expands out from the mean.
